    - analyzeProposal intent: calls scout.discoverProposals then analyzer.evaluateProposal on the matching proposal, returns success with evaluation summary
    - fundProject intent: calls governance.executeFundingPipeline with the specific proposal query and returns success with tx signature info
    - checkTreasury intent: calls treasury.getBalance() and returns success with formatted balance string
    - Unknown intent: returns { success: false, intent, message: 'Unknown command' }
    - Agent method throws: returns { success: false, intent, message: error description }

1. Create `src/voice/voice-types.ts` with:
   - `VoiceIntent` type: `'findProposals' | 'analyzeProposal' | 'fundProject' | 'checkTreasury'`
   - `VoiceCommand` interface: `{ intent: VoiceIntent; params: Record<string, string> }`
   - `VoiceResult` interface: `{ success: boolean; intent: VoiceIntent; message: string; data?: unknown }`

2. Create `src/voice/voice-command-router.ts` with `VoiceCommandRouter` class:
   - Constructor accepts `GovernanceAgent`, `ITreasuryAgent`, and `IScoutAgent` (scout needed for analyzeProposal to find a proposal by query, and for IAnalyzerAgent to evaluate it). Also accept `IAnalyzerAgent`.
   - Actually, simplify: constructor accepts `{ governance: GovernanceAgent, scout: IScoutAgent, analyzer: IAnalyzerAgent, treasury: ITreasuryAgent }` as a single options object. This matches the research pattern but adds scout+analyzer for individual intents.
   - `execute(command: VoiceCommand): Promise<VoiceResult>` -- switch on intent, wrap each handler in try/catch returning `{ success: false }` on error.
   - `handleFindProposals(params)`: call `governance.executeFundingPipeline({ query: params.query || 'new grant proposals', budget: 10000 })`. Format message: "Found N proposals. [summary]". Return data as the DecisionSummary.
   - `handleAnalyzeProposal(params)`: call `scout.discoverProposals(params.proposalId || 'proposals')`, find proposal matching params.proposalId (or take first), call `analyzer.evaluateProposal(proposal)`. Format message with score and recommendation. Return Evaluation as data.
   - `handleFundProject(params)`: call `governance.executeFundingPipeline({ query: params.proposalId || 'fund project', budget: Number(params.amount) || 10000 })`. Format message with allocations and tx info. Return DecisionSummary as data.
   - `handleCheckTreasury()`: call `treasury.getBalance()`. Format message: "Treasury holds X SOL, Y USDC (total ~$Z). N LP positions active." Return TreasuryBalance as data.

3. Create `src/voice/index.ts` barrel exporting from voice-types.ts and voice-command-router.ts.

   - Test error handling: mock throwing, verify success: false returned.
   - Test unknown intent returns success: false.
   - Follow existing test patterns (vi.mock for keys/connection not needed here since we inject everything).

1. Create `src/voice/text-parser.ts` with:
   - `parseTextCommand(text: string): VoiceCommand` function.
   - Keyword-based intent matching (case-insensitive):
     - `find` + (`proposal` | `grant`) => `findProposals`, params: `{ query: text }`
     - `analyze` | `evaluate` => `analyzeProposal`, params: `{ proposalId: extractId(text) }` where extractId takes the word after "proposal" or "analyze"
     - `fund` | `approve` => `fundProject`, params: `{ proposalId: extractId(text), amount: extractAmount(text) }` where extractAmount finds first numeric string
     - `treasury` | `balance` | `check` => `checkTreasury`, params: `{}`
     - Default: `findProposals` with `{ query: text }`
   - Helper `extractId(text)`: find the word after keywords like "proposal", "project", or return the last capitalized/alphanumeric token.
   - Helper `extractAmount(text)`: regex for dollar amounts or plain numbers, return as string.

   - Follow the server factory pattern from Phase 6 (createScoutServer): returns `{ app, start }`.

   - Express app with:
     - `POST /api/voice/command` -- accepts `{ text: string }`, calls `parseTextCommand(text)`, then `router.execute(command)`, returns VoiceResult JSON. Returns 400 if no text provided.
     - `GET /api/voice/signed-url` -- reads `ELEVENLABS_API_KEY` and `ELEVENLABS_AGENT_ID` from env.
