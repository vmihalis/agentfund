1. Create `src/lib/keys.ts` (THE bridge -- per RESEARCH.md Pattern 1):
   - Import `Keypair` from `@solana/web3.js`
   - Import `createSignerFromKeypair` from `@metaplex-foundation/umi`
   - Import `fromWeb3JsKeypair` from `@metaplex-foundation/umi-web3js-adapters`
   - Import `getUmi` from `./metaplex/umi.js`
   - Import `AgentRole, AGENT_ROLES` from `../types/agents.js`
   - Re-export `AgentRole`
   - `loadKeypairBytes(role)`: reads `keys/{role}.json`, parses JSON, returns Uint8Array
   - `getWeb3Keypair(role)`: returns `Keypair.fromSecretKey(loadKeypairBytes(role))`
   - `getUmiSigner(role)`: converts web3 Keypair to Umi KeypairSigner via `fromWeb3JsKeypair` then `createSignerFromKeypair`
   - `getAllWeb3Keypairs()`: returns Record<AgentRole, Keypair> for all 4 agents
   - This is the ONLY file that imports from both Umi and web3.js

2. Create `src/lib/metaplex/umi.ts` (per RESEARCH.md Pattern 2):
   - Import `createUmi` from `@metaplex-foundation/umi-bundle-defaults`
   - Import `mplCore` from `@metaplex-foundation/mpl-core`
   - Import plugin from `@metaplex-foundation/mpl-agent-registry` (check actual export name -- try `mplAgentIdentity` or `mplAgentRegistry`)
   - Import `signerIdentity` from `@metaplex-foundation/umi`
   - Singleton pattern with `_umi` variable
   - `getUmi()`: creates Umi with RPC URL from `process.env.SOLANA_RPC_URL` (fallback `https://api.devnet.solana.com`), registers mplCore() and agent registry plugin
   - `setUmiIdentity(signer)`: sets Umi identity for signing
   - Load dotenv at module level: `import 'dotenv/config'`

3. Create `src/lib/metaplex/collection.ts`:
   - Import from `@metaplex-foundation/mpl-core`: `createCollection`, `fetchCollection`
   - Import `generateSigner` from `@metaplex-foundation/umi`
   - `createAgentCollection(umi)`: creates "AgentFund Agents" collection using generateSigner for collection address. URI as data URI with collection metadata JSON. Calls `sendAndConfirm(umi)`. Returns the collection signer (caller saves the public key). Logs collection address.
   - `fetchAgentCollection(umi, address)`: fetches and returns collection data

4. Create `src/lib/metaplex/agent-nft.ts`:
   - Import from `@metaplex-foundation/mpl-core`: `create`, `fetchAsset`, `fetchCollection`
   - Import `generateSigner`, `publicKey` from `@metaplex-foundation/umi`
   - `createAgentAsset(umi, { name, uri, owner, collectionAddress })`: creates Core Asset in collection. Uses `generateSigner(umi)` for asset address. Sets `owner` to the agent's wallet pubkey (passed as Umi PublicKey string). Fetches collection first for the `collection` parameter. Calls `sendAndConfirm(umi)`. Returns asset signer. Verifies with `fetchAsset` after creation.
   - Note: The Umi identity (deployer) must be the collection authority to add assets

5. Create `src/lib/metaplex/identity.ts`:
   - Import from `@metaplex-foundation/mpl-agent-registry`: `registerIdentityV1`, `findAgentIdentityV1Pda`, `fetchAgentIdentityV1`, `safeFetchAgentIdentityV1`
   - `registerAgentIdentity(umi, { assetAddress, collectionAddress, agentRegistrationUri })`: calls `registerIdentityV1` with asset, collection, and agentRegistrationUri (REQUIRED per RESEARCH pitfall 2). Calls `sendAndConfirm(umi)`. Returns PDA address.
   - `isAgentRegistered(umi, assetAddress)`: derives PDA via `findAgentIdentityV1Pda`, calls `safeFetchAgentIdentityV1`, returns boolean (null = not registered)
   - `verifyAgentIdentity(umi, assetAddress)`: full verification -- derives PDA, fetches identity, checks identity.asset matches assetAddress, fetches Core Asset to verify AppData plugin binding. Returns `{ verified: boolean; pda: string; identity: object | null }`

6. Create `src/lib/metaplex/index.ts` barrel:

   - `createAgentTokenAccount(connection, payer, agentPubkey, mint?)`: creates ATA for agent. Defaults mint to DEVNET_USDC_MINT. Returns ATA info.
