  <action>
1. Create `src/lib/solana/airdrop.ts`:
   - `airdropSol(connection, publicKey, amountSol)`: wraps `connection.requestAirdrop()` with retry logic (3 attempts, exponential backoff starting at 2s). Confirms transaction after airdrop. Returns boolean success.
   - `ensureMinBalance(connection, publicKey, minSol)`: checks balance, airdrops if below minimum. Logs current balance.
   - Update `src/lib/solana/index.ts` to re-export from airdrop.ts

2. Create `scripts/fund-wallets.ts`:
   - Load all 4 agent keypairs via `getAllWeb3Keypairs()` from keys.ts
   - Also load a "deployer" keypair: check if `keys/deployer.json` exists. If not, generate one and save it (same pattern as generate-keys.ts). The deployer is the collection authority and fee payer.
   - For each wallet (deployer + 4 agents = 5 wallets):
     a. Check SOL balance via `connection.getBalance()`
     b. If < 2 SOL, attempt `airdropSol()` for 2 SOL
     c. Log balance after attempt
   - After SOL funding, create Associated Token Accounts for USDC:
     a. Try creating ATAs using `DEVNET_USDC_MINT` first
     b. If the official devnet USDC mint does not exist or ATAs fail to create, fall back to DEMO_USDC:
        - Call `createDemoUSDCMint()` to create custom token
        - Save the mint address to `keys/demo-usdc-mint.json` for reuse
        - Create ATAs for all 4 agents using the DEMO_USDC mint
        - Mint 1000 DEMO_USDC (1_000_000_000 raw, 6 decimals) to each agent's ATA
     c. Log ATA addresses and balances
   - Save a `keys/addresses.json` file with all public keys and ATA addresses for reference:
     ```json
     {
       "deployer": "...",
       "agents": {
         "scout": { "publicKey": "...", "ata": "..." },
         ...
       },
       "usdcMint": "...",
       "isDemoUSDC": true/false
     }
     ```
   - Add script to package.json: `"fund-wallets": "tsx scripts/fund-wallets.ts"`
   - IMPORTANT: This script requires a live devnet connection. Handle RPC errors gracefully with clear error messages about rate limits.

3. Create `scripts/register-agents.ts`:
   - This is THE critical registration script. It runs as a one-time setup (per CONTEXT.md locked decision).
   - Load deployer keypair from `keys/deployer.json`
   - Load all 4 agent keypairs
   - Initialize Umi and set deployer as identity (deployer is collection authority)

   Step A - Create collection (idempotent):
   - Check if `keys/collection.json` exists (contains saved collection address from prior run)
   - If exists, fetch collection from devnet to verify it's still there
   - If not exists or fetch fails, create new collection via `createAgentCollection(umi)`
   - Save collection address to `keys/collection.json`

   Step B - Create agent Core Assets (idempotent):
   - For each role in AGENT_ROLES:
     a. Check if `keys/assets.json` has an entry for this role
     b. If exists, verify asset on devnet via `fetchAsset()`
     c. If not exists or fetch fails, create via `createAgentAsset(umi, { name, uri, owner, collectionAddress })`
        - `name`: from AGENT_CONFIGS[role].name
        - `uri`: data URI with JSON metadata: `data:application/json,${encodeURIComponent(JSON.stringify({ name, role, description }))}`
        - `owner`: agent's wallet pubkey (converted to Umi PublicKey via `publicKey(agentKeypair.publicKey.toBase58())` -- note: this conversion is acceptable in a script, NOT in library code)
        - `collectionAddress`: from keys/collection.json
     d. Save asset address to `keys/assets.json` under role key
   - CRITICAL: Use separate transactions per asset (per RESEARCH pitfall 4 -- transaction size limits)

   Step C - Register identities (idempotent):
   - For each role in AGENT_ROLES:
     a. Load asset address from `keys/assets.json`
     b. Check if already registered via `isAgentRegistered(umi, assetAddress)`
     c. If not registered, call `registerAgentIdentity(umi, { assetAddress, collectionAddress, agentRegistrationUri })`
        - `agentRegistrationUri`: data URI: `data:application/json,${encodeURIComponent(JSON.stringify({ name, role, version: "1.0.0", capabilities: [...] }))}`
     d. Verify registration via `verifyAgentIdentity(umi, assetAddress)`
     e. Log PDA address
   - CRITICAL: Each registration is a separate transaction

   Step D - Summary:
   - Log all 4 agents with: role, wallet pubkey, asset address, PDA address, verification status
   - Save complete state to `keys/registration.json`:
     ```json
     {
       "collection": "...",
       "agents": {
         "scout": { "wallet": "...", "asset": "...", "pda": "...", "verified": true },
         ...
       }
     }
     ```

   - Add to package.json scripts: `"register-agents": "tsx scripts/register-agents.ts"`
   - Add convenience script: `"setup": "tsx scripts/generate-keys.ts && tsx scripts/fund-wallets.ts && tsx scripts/register-agents.ts"`
  </action>
