    2. **dashboard/src/app/api/voice/signed-url/route.ts:** GET handler. Proxy to voice server: `fetch('http://localhost:${process.env.VOICE_SERVER_PORT || 4003}/api/voice/signed-url')`. Return the { signedUrl } response. On failure, return 503 with { error: 'Voice server unavailable' }.

    3. **dashboard/src/app/api/voice/command/route.ts:** POST handler. Accept { text: string } body. Proxy to voice server: `fetch('http://localhost:${process.env.VOICE_SERVER_PORT || 4003}/api/voice/command', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ text }) })`. Return the VoiceResult response. On failure, return 503.

    4. **dashboard/src/lib/proposals-store.ts:** Create a shared in-memory proposals store module. This module is the SINGLE SOURCE OF TRUTH for proposals data -- both `/api/proposals` and `/api/proposals/submit` (Plan 08-03) import from here.

    Export:
    - `proposals: PipelineProposal[]` -- mutable array holding all proposals (seeded with 3 demo entries)
    - `addProposal(proposal: PipelineProposal): void` -- pushes a new proposal onto the array
    - `getProposals(): PipelineProposal[]` -- returns the current proposals array

    Seed with 3 demo proposals in different stages to demonstrate the pipeline visualization:
    - "AI Code Review Tool" at stage 'evaluating'
    - "DeFi Yield Optimizer" at stage 'submitted'
    - "Cross-chain Bridge Monitor" at stage 'approved'

    IMPORTANT: This shared store is what makes proposal submission (Plan 08-03) visible in the pipeline (this plan). Plan 08-03's `/api/proposals/submit/route.ts` will import `addProposal` from this module to push new submissions into the same array.

    5. **dashboard/src/app/api/proposals/route.ts:**
    - GET: Import `getProposals` from `@/lib/proposals-store`. Return the proposals array as JSON.
    - POST: Accept { text: string } and proxy to voice command endpoint to trigger pipeline actions. Parse the voice result to update the in-memory pipeline state. This lets the dashboard trigger pipeline actions and see results.

    6. **dashboard/src/lib/types.ts:** Add types:
    - `VoiceResult`: { success: boolean, intent: string, message: string, data?: unknown }
    - `PipelineProposal`: { id: string, title: string, stage: 'submitted'|'evaluating'|'approved'|'funded', updatedAt: number, evaluation?: { overallScore: number, recommendation: string, reasoning: string } }
    - `PipelineStage`: 'submitted' | 'evaluating' | 'approved' | 'funded'

    7. **dashboard/src/lib/api.ts:** Add helpers:
    - `fetchSignedUrl(): Promise<{ signedUrl: string }>` -- GET /api/voice/signed-url
    - `sendCommand(text: string): Promise<VoiceResult>` -- POST /api/voice/command
    - `fetchProposals(): Promise<PipelineProposal[]>` -- GET /api/proposals

    8. **tests/unit/dashboard-proposals.test.ts:** Test the pipeline stage mapping logic. Create a pure function `mapPipelineStage(step: string, status: string): PipelineStage` that maps backend pipeline events to dashboard stages:
    - discover:started -> submitted
    - evaluate:started -> evaluating
    - evaluate:completed (with recommendation 'fund') -> approved
    - fund:completed -> funded
    Test this function with each mapping. Also test the PipelineProposal shape has required fields.
