  </behavior>
  <action>
    1. Install Express: `pnpm add express @types/express`

    2. Create `src/lib/x402/types.ts` with these interfaces:
       - `X402Config`: { recipientWallet: PublicKey, usdcMint: PublicKey, priceUsdc: number (base units, 1 USDC = 1_000_000), connection: Connection, cluster: 'devnet' | 'mainnet-beta' }
       - `PaymentRequirements`: the 402 response body shape { x402Version: 1, scheme: 'exact', network: string, payment: { recipientWallet: string, tokenAccount: string, mint: string, amount: number, amountUSDC: number } }
       - `PaymentProof`: the X-Payment header decoded shape { x402Version: 1, scheme: 'exact', network: string, payload: { serializedTransaction: string } }
       - `VerifyResult`: { valid: boolean, signature?: string, error?: string }
       - `WrapFetchOptions`: { keypair: Keypair, connection: Connection, usdcMint: PublicKey, maxPaymentUsdc?: number }

    3. Create `src/lib/x402/verify.ts` implementing `verifyAndSettlePayment(connection, serializedTx, expectedRecipientAta, expectedMinAmount)`:
       - Deserialize base64 transaction with Transaction.from(Buffer.from(serializedTx, 'base64'))
       - Iterate tx.instructions looking for TOKEN_PROGRAM_ID instructions where data[0] === 3 (Transfer)
       - Read amount as BigUInt64LE from data offset 1
       - Check keys[1] (destination) matches expectedRecipientAta
       - Check amount >= expectedMinAmount
       - If no valid transfer found, return { valid: false, error: 'Invalid transfer instruction' }
       - If valid, simulate transaction (connection.simulateTransaction), return error if sim fails
       - Submit with sendRawTransaction (skipPreflight: false, preflightCommitment: 'confirmed')
       - Await confirmTransaction(signature, 'confirmed')
       - Return { valid: true, signature }
       - Follow the exact pattern from 06-RESEARCH.md "Transaction Verification (Server-Side)" code example

    4. Create `src/lib/x402/index.ts` barrel exporting all public symbols from types.ts and verify.ts (middleware and client exports will be added in Task 2).

    5. Create `tests/unit/x402-verify.test.ts` covering the three behavior cases above:
       - Mock @solana/web3.js Transaction.from to return a transaction with controlled instructions
       - Mock connection.simulateTransaction and connection.sendRawTransaction and connection.confirmTransaction
       - Test rejection when no SPL transfer instruction matches the expected recipient ATA
       - Test rejection when transfer amount is below expectedMinAmount
       - Test successful extraction of destination and amount from a properly formed transfer instruction, through to simulated settlement
       - Use vi.mock and vi.hoisted() per project Vitest 4 pattern
  </action>

    - Test: wrapFetchWithPayment passes through non-402 responses unchanged

    - Test: wrapFetchWithPayment throws when payment amount exceeds maxPaymentUsdc safety cap

  <action>
    1. Create `src/lib/x402/middleware.ts`:
       - Export `x402Middleware(config: X402Config)` returning Express middleware `(req, res, next)`
       - Compute recipientAta via getAssociatedTokenAddressSync(config.usdcMint, config.recipientWallet) at middleware creation time (not per-request)
       - If no X-Payment header: respond 402 with PaymentRequirements JSON including recipientWallet, tokenAccount (ATA), mint, amount, amountUSDC (amount / 1_000_000)
       - If X-Payment header present: decode from base64 to JSON, parse as PaymentProof, call verifyAndSettlePayment with serializedTransaction, recipientAta, priceUsdc
       - On valid: attach signature to `(req as any).x402Signature = result.signature`, call next()
       - On invalid: respond 402 with error message

    2. Create `src/lib/x402/client.ts`:
       - Export `wrapFetchWithPayment(baseFetch, options: WrapFetchOptions)` returning a new fetch function
       - The wrapped fetch: call baseFetch first
       - If response.status !== 402: return response as-is
       - If 402: parse response JSON as PaymentRequirements
       - Safety check: if options.maxPaymentUsdc set and amount > maxPaymentUsdc, throw Error with details
       - Build SPL transfer: get payer ATA via getAssociatedTokenAddressSync(options.usdcMint, options.keypair.publicKey), dest ATA from requirements.payment.tokenAccount
       - Create Transaction with feePayer = options.keypair.publicKey, fresh blockhash from connection.getLatestBlockhash()
       - Add createTransferInstruction(payerAta, new PublicKey(destAta), options.keypair.publicKey, amount)
       - Sign with tx.sign(options.keypair)
       - Encode PaymentProof as JSON, then base64: Buffer.from(JSON.stringify(paymentProof)).toString('base64')
       - Retry original request with X-Payment header added
       - Return the retry response

    3. Update `src/lib/x402/index.ts` to also export x402Middleware from middleware.ts and wrapFetchWithPayment from client.ts.

    4. Create/update test files:

       - `tests/unit/x402-client.test.ts`: Mock baseFetch to return 402 with payment requirements JSON then 200. Mock connection.getLatestBlockhash. Test passthrough on non-402, payment creation on 402, safety cap enforcement.

    Note: Use vi.mock for @solana/spl-token (getAssociatedTokenAddressSync, createTransferInstruction) and for verify.ts. Use vi.hoisted() for mock functions per project Vitest 4 pattern. Mock connection methods to avoid network calls.
