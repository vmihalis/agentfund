Create `src/lib/unbrowse/client.ts`:
- `UnbrowseClient` class with constructor accepting `baseUrl` (default `http://localhost:6969`) and `timeoutMs` (default 15000)
- `resolveIntent(intent: string, targetUrl?: string): Promise<unknown>` -- POST to `/v1/intent/resolve` with AbortController timeout. Body: `{ intent, params: targetUrl ? { url: targetUrl } : undefined, context: targetUrl ? { url: targetUrl } : undefined }`. Throws on non-OK status.
- `healthCheck(): Promise<boolean>` -- GET to `/health` with 3-second AbortSignal.timeout. Returns true/false, never throws.
- Support optional `UNBROWSE_API_KEY` env var as Bearer token if set.

Create `src/lib/unbrowse/parser.ts`:
- Flexible Zod schema (`UnbrowseProposalSchema`) accepting multiple field name variations: title/name, description/summary, amount/requested_amount/funding_amount (number or string), team/team_info, url/link
- `extractResultArray(raw: unknown): unknown[]` helper that checks `raw.result` (array or object), `raw.data`, direct array, wraps single object in array
- `parseAmount(val: unknown): number` helper converting string amounts (strip $ and commas) to numbers, defaulting to 0
- `parseUnbrowseResult(raw: unknown): Proposal[]` -- extracts array, validates each item with Zod safeParse, normalizes field names, generates IDs as `unbrowse-{timestamp}-{index}`, filters out entries with empty title
- Use `import { z } from 'zod'` (Zod v4 already installed)

Create `src/agents/scout-agent.ts`:
- `ScoutAgent extends BaseAgent implements IScoutAgent`
- Constructor: `(bus: AgentEventBus, unbrowseClient?: UnbrowseClient)` -- injects client for testability; defaults to `new UnbrowseClient(process.env.UNBROWSE_URL ?? 'http://localhost:6969')`
- Private `cache: Proposal[]` array, initially empty
- `initialize()`: calls `this.unbrowseClient.healthCheck()`, emits status 'initialized' with health result
- `shutdown()`: emits status 'shutdown', no-op otherwise
- `discoverProposals(query: string)`:
  - Emits status 'discovering' with query
  - Imports `GRANT_TARGETS` from unbrowse lib
  - For each target, tries `unbrowseClient.resolveIntent(query + ' ' + target.intent, target.url)` inside try/catch
  - Parses each result with `parseUnbrowseResult`, collects all proposals
  - Deduplicates by title (case-insensitive, keep first seen)
  - If proposals found: cache them, emit 'unbrowse-resolved', return
  - If no proposals from Unbrowse: check cache, if non-empty emit 'using-cache', return cache
  - If cache also empty: emit 'using-stub', return `STUB_PROPOSALS` (imported from `./stubs/stub-scout.js` -- extract the constant)
- Export `STUB_PROPOSALS` from `src/agents/stubs/stub-scout.ts` so ScoutAgent can import it as fallback data. Currently the proposals array is inline in the `discoverProposals` method -- refactor to a named export `STUB_PROPOSALS` and have both `StubScoutAgent` and `ScoutAgent` reference it.
